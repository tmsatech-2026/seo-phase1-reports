Project Overview

TMSA SEO Automation
System

A Claude Code-powered SEO audit and optimisation system for South African tourism properties. Three workflows - leads pipeline (full), leads pipeline (quick batch), and client delivery. All produce HTML reports and write structured data to Google Sheets.

3 active workflows 31+ prospects in pipeline Platform: Claude Code + Playwright + Google Sheets Hosting: Cloudflare Pages

What This Project Does

Three distinct workflows serve two purposes - lead generation for new prospects, and deep SEO delivery for paying clients.

Workflows 1a + 1b

Leads Audit Pipeline

Finds tourism properties running Google Ads, crawls their website, and produces a free SEO audit report published live on Cloudflare Pages. Two modes: Quick Cycle (batch-process the entire queue, you decide which sites get a Gmail) and Full Pipeline (single site, end-to-end including Gmail draft - use when you are ready to send immediately).

Workflow 2

Client SEO Audit (Phase 1-4)

A multi-phase SEO engagement for paying clients. Phase 1 crawls and audits the site. Phase 2-3 generates optimised titles, descriptions and H1s for every page. Phase 4 produces a before-and-after summary report. All data lives in the client's own Google Sheet for full transparency.

Target sites for both workflows: Independent South African guesthouses, boutique hotels, safari lodges, and B&Bs with 8-30 pages. Group/chain hotel sites (20+ properties) are not suitable - skip these.

Leads Audit Pipeline

Prospect finds a tourism property running Google Ads - adds URL to queue - full audit runs automatically - Gmail draft created ready to send.

How to start this workflow
"Run the Leads Audit Pipeline - follow WORKFLOW-LEADS-AUDIT.md" "Process [domain] from the Audit Queue using the full Leads Audit Pipeline"
Workflow doc: WORKFLOW-LEADS-AUDIT.md - runs all 9 steps including Cloudflare publish and Gmail draft.
1
Add URL to Audit Queue (Google Sheet)

Open the master Google Sheet and add the property URL in Column A. Set Status = Pending. That is the only manual step.

2
Read the Queue

Claude reads the Audit Queue tab and finds all Pending entries. Creates a numbered client folder (08-landrani/, 09-leschambres/, etc.) and marks the row In Progress.

3
Crawl the Website (Playwright)

Breadth-first crawl from homepage. Extracts page title, meta description, H1, slug and page type for every page. Also extracts all email addresses found on the site and identifies the best hero image for the report header. Saves to pages_data.json.

4
Write Crawl Data to Google Sheet

Crawl data is written to a new tab in the master sheet named after the domain (e.g. landrani.com). Full metadata preserved - no truncation.

5
Analyse Issues and Calculate Health Score

From the crawl data: counts short titles (< 50 chars), missing descriptions, descriptions > 160 chars, missing H1s. Calculates weighted health score per page (Title 50pts + Desc 35pts + H1 15pts). Site score = average across all pages. Identifies the top 3 most impactful issues for this specific site.

6
Generate HTML Audit Report

Populates the master HTML template with all client-specific variables. Includes 4 KPI cards, a critical issues section, Google Ads callout, 3 issue cards (customised per site), full page-by-page table with tooltips, and a green CTA section. Saved to the client's output folder.

7
Update Audit Queue + Publish to Cloudflare Pages

Four sub-steps run in sequence: (7a) Mark Report Ready + write health score to column C. (7b) Strip tracking params from URL in column A, leaving clean base URL. (7c) Copy HTML report to GitHub repo (tmsatech-2026/seo-phase1-reports) and push - Cloudflare auto-deploys within 60 seconds. (7d) Write live Cloudflare URL to column D. Live URL: seo-phase1-reports.pages.dev/clients/[slug]/seo-audit-report.html

8
Draft Gmail Outreach Email

Gmail draft created via MCP. To: first email found on site. CC: any additional emails. Subject: "We audited [domain] - here's what we found". HTML email with issue summary, live report link, and Google Ads connection callout. Saved as draft - reviewed and sent manually.

9
Mark Complete

Audit Queue row updated to Complete. Workflow moves to next Pending entry in the queue.

Workflow 1c - Email Cycle (Batch Gmail Drafts)

Runs after you have reviewed the 1b Quick Cycle reports and marked chosen rows "Draft Email". Batch-drafts a personalised Gmail outreach email for every marked row, pulling the top 3 issues from each site's crawl data. Marks each row "Email Drafted" when done.

Leads Audit Email Cycle

Reads all "Draft Email" rows from the Audit Queue. For each row, derives the client folder slug from the Cloudflare URL in col D, reads the live HTML report as the source of truth, extracts the top 3 issue cards and critical-impact paragraph, then creates a Gmail draft. Report content drives the email - not pages_data.json.

How to start this workflow
"Run the Leads Audit Email Cycle - follow WORKFLOW-LEADS-AUDIT-EMAIL-CYCLE.md"
Workflow doc: WORKFLOW-LEADS-AUDIT-EMAIL-CYCLE.md - processes every "Draft Email" row. Pre-condition: rows must be manually set to Draft Email after reviewing 1b reports.
1
Read the Queue - All "Draft Email" Entries

Reads the Audit Queue tab and filters for rows with status === "Draft Email". If none found, stops. Logs the list of domains to be processed before starting.

2
Locate the HTML Report

For each row: reads URL (col A), Cloudflare URL (col D), and emails (col E). Parses the client folder slug from the Cloudflare URL (e.g. 21-rustygate). Globs for the *.html audit report in the client's output/ folder. This HTML file is the single source of truth for all email content.

3
Extract Content from the HTML Report

Reads the .issue-card elements directly from the report HTML - extracting .stat-line, h3, and the first <p> from each of the 3 cards. Synthesises the bold summary sentence from the .critical-impact paragraph. Client name from the hero <h1>. No recalculation - what's in the report is what goes in the email.

4
Draft Gmail Email via MCP

Creates a Gmail draft via the Gmail MCP. Subject: "We built a free SEO report for [domain] - thought you should see it...". HTML email with 3 issue bullets drawn from the report cards, a CTA button to the live report with UTM tracking params (?utm_source=email&utm_medium=direct&utm_campaign=seo-outreach&utm_content=[slug]), and a Google Ads Quality Score callout box. All inline styles - Gmail strips style blocks.

5
Mark "Email Drafted" + Move to Next

Updates the Audit Queue row to Email Drafted. Immediately processes the next "Draft Email" entry. When all are done, prints a summary of drafts created.

After the Email Cycle: Open Gmail Drafts folder. Review and personalise each email if needed. Send. Then manually set each sent row in the Audit Queue to Complete.

Workflow 2 - Client SEO Audit (Phase 1-4)

For paying clients. A structured multi-phase engagement. Each phase requires client sign-off before proceeding. Deeper analysis than the leads audit - includes actual SEO recommendations written directly into their Google Sheet.

Phase 1 - Website Crawl and Baseline Report

Crawl the client's live site, write data to their dedicated Google Sheet, generate a Phase 1 HTML report. Stop here and wait for client to verify the data before any further work.

Technology Stack

All tools run inside Claude Code sessions. No separate server or infrastructure required.

Playwright MCP

Headless browser crawling. Extracts live metadata from every page including JavaScript-rendered content.

Google Sheets API

All crawl data and recommendations written to sheets. Clients can view, edit and approve in a familiar interface.

Gmail MCP

Creates outreach email drafts programmatically. HTML emails with issue summaries and live report links. Sent manually after review.

Cloudflare Pages

HTML reports hosted at a live URL via GitHub auto-deploy. Reports are live within 60 seconds of a git push. Professional shareable links.

Pipeline Intelligence — What's Been Built

Key improvements implemented since launch. All baked into workflow docs and templates — no manual steps needed.

Health Score
Weighted Scoring Model

Per-page score: Title (50pts) + Meta Description (35pts) + H1 (15pts) = 100 max. Site score = average across all pages. Bands: Good 80-100% / Needs Work 60-79% / Poor 40-59% / Critical 0-39%. Replaces the old binary pass/fail ratio. Score and tooltip in report now match.

Accuracy
Zero-SEO Detection + Accurate SERP Framing

Detects when all pages share identical descriptions or match known theme placeholders (BeHotel2, "Just another WordPress site"). Switches to accurate "lost control" framing - Google ignores useless descriptions and generates its own snippets. Never say "Google shows [placeholder] in search results" - it does not.

Email Cycle
HTML Report as Source of Truth

Email Cycle now reads the live HTML report - not pages_data.json. Issue bullets are extracted directly from .issue-card elements. What the client sees in the report is exactly what's in the email. No drift, no recalculation, no new claims.

Analytics
Report Link Tracking

Cloudflare Web Analytics enabled on seo-phase1-reports.pages.dev. CTA button in every email includes UTM params: ?utm_source=email&utm_medium=direct&utm_campaign=seo-outreach&utm_content=[slug]. Filter by path in Cloudflare Analytics to see per-client report visits.

Report UX
Cleaner Issue Cards + Health Score Tooltip

"Why it matters" sections removed from all issue cards - too much text, distracts from the finding. Health score KPI card now has a hover tooltip explaining the weighted calculation. KPI card formatting fixed: score and band label correctly separated across all reports.

Email Copy
Copy and Tone Standards

No em-dashes anywhere - hyphens only. "We ran a detailed SEO Audit of" (not "crawl"). "Tourism Marketing SA" signature throughout. "3 SEO Opportunities Found" (not "critical issues"). Issues framed as opportunities, never problems. Bullets match report exactly - no new claims, no different numbers.
